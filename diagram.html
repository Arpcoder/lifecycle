Define

This is the information to support the diagram, roles, and stages of the API lifecycle above.

  • Lifecycle
    • Business Requirements - Providing the requirements for the API, based upon what the business needs.
    • Security Requirements - Providing the security requirements to esnure it is a high priority.
    • Workspace - Setting up a private, team, partner, or public workspace for working with the API.
    • Source Control - Setting up the source control for the API code, artifacts, and pipelines.
  • Role
    • Product Manager - A product manager is the helping define what is needed for this API.
    • Architect - An architect is part of the conversaiton to help with the big picture of the API.
  • Tooling
    • Confluence - A confluence wiki is used to manage the business and security requirements.
    • Postman - Postman is used to provide the workspace and platform for managing the API.
    • Github - GitHub is used for the repository, requirements, and the feedback loop.
  • Governance
    • Guidelines - Offering operational level guidance for teams to use when delivering the API, helping teams be more consistent in their work.
    • Spectral Rules - Defining which rules are applicable to an API, helping select the right patterns needed applied across the APIs being delivered.
    • Standards - Providing all of the standards that should be applied when it comes to designing and delivering an API, help standardize API offerings.
    • Templates - Providing a set of reusable templates that team members can use when it comes to designing and delivering the API more quickly.
