API Lifecycle Blueprints

This is a proof of concept to see how we can define the API lifecycle in a modular and flexible enough way that anyone can define their own version of the API lifecycle using a common set of elements and actions. This runs on Github, and everything is defined using YAML, allowing it to be forked and worked on collaboratively, using Github issues to evolve and iterate upon the repository of API lifecycle blueprints, elements, and actions.

Structure of this Project

This project takes full advantage of Jekyll and Github for the structure of the project, allowing it to be a living, evolvable catalog of information for the API lifecycle, leveraging the following objects.

Blueprints

Common and custom blueprints that are defined as YAML and can be iterated and evolved upon to define a specific API lifecycle, but also provide ready to go blueprints that others can follow.

  • Name - The name of the blueprint
  • Description - A description of the blueprint.
  • Image - An image to represent the blueprint.
  • Tags - One or more tags to define the blueprint.
  • Stage - What stage of the lifecycle a blueprint applies.
  • Type - Whether a blueprint applies to synchronous or asynchronous APIs.
  • Order - The order of the blueprints when listed out.
  • Areas - The different areas of an API lifecycle.
    • Label - A label for the this area of the lifecycle.
    • Description - A description for the this area of the lifecycle.
    • Elements - The multiple elements that make up this area of the lifecycle.
  • Body - The full body description for this API lifecycle blueprint.

Elements

A mix bag of elements that can be used address all areas of the API lifecycle, providing all of the common elements API producers are using to define and manage their API operations.

  • Name - The name of the element
  • Description - A description of the element.
  • Links - Supporting links for this element.
    • Title - The title of the element link.
    • URL - The description of the element link.
  • Video - A video overview for users too learn about this element.
  • Body - The full body description for this API lifecycle element.
  • Actions - Actions that are possible involving an element.
  • Screenshots - Platform screenshot supporting this element.
    • Title - The title of the element screenshot.
    • URL - The description of the element screenshot.

Actions

A variety of actions that can be taken in associated with the various elements of the API lifecycle, providing simple, yet flexible ways that different stakeholders can take action across the API lifecycle.

  • Name - The name of the action.
  • Description - A description of the action.
  • Links - Supporting links for this action.
    • Title - The title of the action link.
    • URL - The description of the action link.
  • Video - A video overview for users too learn about this action.
  • Body - The full body description for this API lifecycle action.
  • Tools - Open source tooling to power each action.
    • Title - The title of the action tool.
    • Description - The description of the action tool
    • URL - The description of the action tool.
  • Screenshots - Platform screenshot supporting this action.
    • Title - The title of the action screenshot.
    • URL - The description of the action screenshot.
